# Team Organization Note Templates

These Markdown files are examples for `nb` coordination notes. Copy the shape
that matches a project's team topology into that project's notebook, then adapt
the placeholders and link the resulting note from the project's `AGENTS.md`
`Project Notes` section.

Suggested use:

1. Choose the closest topology template.
2. Create a note under `coordination/general` or a lane-specific coordination
   folder.
3. Replace placeholder role names, Agentmux IDs, branch names, and signoff
   rules.
4. Keep stable policy in the org note. Put staffing churn in living-roster
   notes and routine status in rolling handoffs.

Templates:

- `solo-owner-xo.md` - owner plus deputy/XO.
- `coordinator-specialists.md` - coordinator/strategist plus peer specialists.
- `two-tier-lanes.md` - coordinator, lane owners, engineers, reviewers, and
  side-car roles.
- `cross-repository-maintainers.md` - one bundle coordinating maintainers of
  multiple mostly independent repositories.
